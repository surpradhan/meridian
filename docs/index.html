What it does

Built for data, not for DBAs

Ask questions in plain English. Get answers, tables, and charts — instantly.

🤖

Multi-Agent Routing

A router agent classifies each query and dispatches it to a specialized Sales, Finance, or Operations agent via LangGraph orchestration.

🔒

Safe by Default

Every query is validated with SQL syntax pre-checking (SQLite EXPLAIN) before touching the database. Injection and destructive operations are blocked automatically.

📊

Interactive Charts

Plotly visualizations are auto-selected per result (line, bar, pie) and rendered inline alongside tabular data in the Gradio UI.

🔭

Production Observability

Prometheus /metrics, OpenTelemetry traces to Jaeger, Grafana dashboards, LangSmith span tracing, and alert rules — all included.

How it works

Five-step pipeline

Every question flows through a deterministic pipeline — routing, specialization, validation, execution, response.

Agents understand context, validators enforce safety, and the database only ever sees clean queries. Results come back as natural language, tabular data, and auto-selected Plotly charts.

Quick Start

Up in 5 minutes

Clone, configure, and run.

Local
# 1. Clone the repo
git clone https://github.com/surpradhan/meridian.git && cd meridian

# 2. Install dependencies
python3 -m venv venv && source venv/bin/activate
make install-dev

# 3. Configure (Groq preferred, OpenAI as fallback)
cp .env.example .env.local
export GROQ_API_KEY="gsk-your-key-here"

# 4. Launch
make ui     # Gradio UI  → http://localhost:7860
make dev    # REST API   → http://localhost:8000/docs
