Why MERIDIAN

Data access, reimagined

Enterprise data is locked behind SQL expertise and rigid dashboards. MERIDIAN breaks down that barrier.

✕ The Problem

  • ✕ Requires SQL expertise to get any answer
  • ✕ Hours of waiting for BI reports
  • ✕ High risk of SQL injection and malformed queries
  • ✕ Limited to pre-built dashboards
  • ✕ No context-awareness across departments

✓ The Solution

  • ✓ Ask questions in plain English — no SQL needed
  • ✓ Get answers in seconds, not hours
  • ✓ Query validation prevents errors by default
  • ✓ Ad-hoc queries across any data domain
  • ✓ Domain-aware agents understand Sales, Finance & Ops
🤖

Multi-Agent Routing

A router agent detects the query domain and dispatches to specialized Sales, Finance, or Operations agents for accurate results.

🔒

Safe by Default

Every query passes through a validator before execution. SQL injection, schema mismatches, and unsafe operations are blocked automatically.

⚡

Instant Answers

Backed by GPT-4, LangChain, and LangGraph — your natural language question becomes a safe database query in milliseconds.

📊

Interactive UI

Gradio-powered web interface lets anyone explore data in tabular format without touching a terminal or writing a line of code.

🔭

Full Observability

Integrated LangSmith tracing and OpenTelemetry support so you can debug, monitor, and optimize every query in production.

🐳

Docker Ready

Single docker-compose up spins up the full stack — API, UI, PostgreSQL — locally or in the cloud.

How it works

Five-step pipeline

Every question flows through a deterministic pipeline — routing, specialization, validation, execution, response.

The architecture is designed for correctness first: agents understand context, validators enforce safety, and the database only ever sees clean, validated queries.

💬
User Question Natural language input from any user
↓
🧭
Router Agent Detects domain: Sales / Finance / Operations
↓
🤖
Domain Agent Handles specialized query construction
↓
🛡️
Query Validator Ensures safety before any DB access
↓
📊
Response Natural language answer + tabular data

Development Roadmap

Phase by phase

MERIDIAN is built in deliberate phases — each one shipping real, working functionality.

Phase 1: Foundation ✅ Complete

View Registry, Sales/Finance/Operations agents, QueryBuilder, QueryValidator, REST API (6 endpoints), Gradio UI, Docker, 158+ tests.

Phase 2: Activate Scaffolded Features (Weeks 1–2)

Wire Redis caching, pagination, rate limiting, OpenTelemetry tracing, and LangSmith integration — all code written, wiring in progress.

Phase 3: LLM-Powered NL Understanding (Weeks 3–5)

Replace regex-based routing with GPT-4 classification, structured output, and confidence-based clarification.

Phase 4: Conversational Intelligence (Weeks 6–8)

Multi-turn conversations, query history, smart follow-up suggestions, LangGraph as primary orchestrator.

Phase 5: Enterprise Security (Weeks 9–12)

JWT auth, row-level access control, audit logging, CORS lockdown.

Phase 6: Advanced Query Capabilities (Weeks 13–16)

Window functions, CTEs, subqueries, multi-hop joins, time intelligence, auto-visualization with Plotly.

Phase 7: Scale & Polish (Weeks 17–20)

Streaming responses, async execution, self-service domain onboarding, export options, load testing.

Quick Start

Up in 5 minutes

Clone, configure, and run — locally or in Docker.

Local
Docker
# 1. Clone the repo
git clone https://github.com/surpradhan/meridian.git
cd meridian

# 2. Create virtual environment
python3 -m venv venv
source venv/bin/activate

# 3. Install dependencies
make install-dev

# 4. Configure environment
cp .env.example .env.local
export OPENAI_API_KEY="sk-your-key-here"

# 5. Run tests
make test-unit

# 6. Launch the UI
make ui     # → http://localhost:7860
#  or the API
make dev    # → http://localhost:8000/docs
